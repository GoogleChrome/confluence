Expectations:
  • Search box with search icon on left
  • Interactive "X" button appears when query non-empty; clicking clears query
  • Pressing Escape when input is focused clears query
  • Default predicate is True
  • Predicate parsed until part is not understood (e.g., "a b ~c" parses to keywords "a" and "b")
  • Supports structured query "in:a" "notin:a" (in/not in browser starting with a); "count:2" (in exactly 2 of selected browsers)
  • Predicate remains unchanged when parse totally fails (e.g., paste in string: "in:notarealbrowser")
